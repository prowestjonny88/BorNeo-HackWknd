# Pasar Memory User Guide

This guide shows a simple daily workflow for a hawker selling **nasi lemak** and **nasi putih**.

## 0. First Login Setup (Required)
1. After first login, open **Menu Setup**.
2. Add your own items and prices before starting sales.
   - Example:
     - `nasi lemak` - `RM 8.00`
     - `nasi putih` - `RM 2.00`
3. Make sure each item is active and saved.

## 1. Start Selling (Tap During Sales)
1. Open the app and go to the **Selling** screen.
2. Make sure your menu has:
   - `nasi lemak`
   - `nasi putih`
3. Every time you sell an item, tap the item card once.
   - Example: If you sell 1 nasi lemak, tap `nasi lemak` one time.
   - If you sell 3 nasi putih, tap `nasi putih` three times.
4. Continue tapping throughout the day.

## 2. Upload Evidence at End of Day
1. Go to the **Evidence** step.
2. Upload your **TNG transaction history** screenshot(s) or file as evidence.
3. Wait for evidence processing to complete.

## 3. Record Voice Recap or Type Recap
1. Go to the **Recap** step.
2. Record voice recap or type your summary.
   - Example: `Today I sold nasi lemak 20, nasi putih 15, cash 200. Most payments were TNG/QR.`
3. Submit recap for analysis.

## 4. Review Recap
1. Go to **Review Recap**.
2. Check detected items and quantities.
3. Check counted cash value.
4. Edit anything that is incorrect.
5. Confirm recap and cash.

## 5. View Daily Ledger and Save
1. Open the **Daily Ledger** screen.
2. Review totals from:
   - Tapped sales
   - Evidence (TNG history)
   - Voice/text recap
   - Counted cash
3. If everything is correct, tap **Save as Today Ledger**.

## Quick Daily Flow
`First login: Menu Setup -> Selling (tap items) -> Evidence upload -> Recap (voice/text) -> Review & Confirm -> Daily Ledger -> Save`

## Tips
- Tap items as you sell to improve end-of-day accuracy.
- Upload clear TNG evidence screenshots.
- Always review detected quantities and cash before saving.
- If detection is wrong, edit in Review Recap before final confirmation.
